# IndustrialEquipmentLogAIAnalyzer
基于 .NET 8 + WPF 开发的**工业设备日志 AI 分析工具**，个人技术作品集 Demo。
项目完整实现「日志导入解析 → 大模型智能故障分析 → 结构化报告导出」业务闭环，兼顾代码可读性、功能可配置性与工业软件使用体验。

---

## 1. 项目简介
本工具面向工厂运维、设备操作、维修工程师，实现工业设备运行日志智能化诊断。
结合自研工业故障知识库与精细化提示词工程，调用大模型 API 完成故障分析，自动生成标准化、可追溯的诊断报告，提升现场故障排查效率。

核心能力：
- 解析标准格式工业日志，识别时间、告警级别、运行信息
- 内置专业诊断知识库与规则化提示词，约束大模型输出，降低 AI 幻觉
- 调用主流大模型 API，输出结构化故障分析结果
- 自动生成 Markdown/TXT 格式诊断报告并支持本地导出

## 2. 使用场景
- 设备日常巡检：对异常日志快速完成故障初判
- 运维工作交接：生成标准化、可追溯的诊断记录
- 设备维修前置：归纳故障诱因、分级给出处置建议
- 技术演示：工业上位机软件 + AIGC 能力集成落地样例

## 3. 技术栈
- 运行平台：.NET 8
- 开发语言：C# 12
- 桌面框架：WPF
- UI 组件：MaterialDesignThemes、MaterialDesignColors
- 序列化/配置：System.Text.Json（UTF-8 BOM 编码，兼容中文）
- 网络请求：原生 HttpClient（异步调用，避免 UI 阻塞）
- 密钥安全：DPAPI(ProtectedData) 本地用户级加密存储密钥

## 4. 运行环境依赖
- 操作系统：Windows 10 / Windows 11
- 开发环境：Visual Studio 2022（安装 WPF 桌面开发组件）
- 运行依赖：.NET 8 SDK / .NET 8 Desktop Runtime
- 外部依赖：可用的大模型 HTTP API（项目兼容 DeepSeek 等主流接口）

## 6. 快速运行步骤
1. 使用 Visual Studio 打开项目解决方案（`.sln`）；
2. 进入 `config/appsettings.json`，配置大模型 API 地址、模型名称；
3. 配置 API 访问密钥（推荐系统环境变量方式）；
4. 按下 `F5` / `Ctrl+F5` 启动程序。

## 7. API Key 配置方式（二选一，优先环境变量）
### 方式一：系统环境变量（推荐，安全性最高）
1. 在 `appsettings.json` 中填写自定义环境变量名称（如 `DEEPSEEK_API_KEY`）；
2. 在 Windows 系统中添加对应**用户级环境变量**，填入真实 API Key；
3. 重启程序即可自动读取密钥。

### 方式二：界面配置+本地加密存储
1. 在软件「大模型设置」区域输入 API Key，点击「保存配置」；
2. 密钥将通过 DPAPI 加密后存入配置文件，界面不再明文回显；
3. 若配置中无有效密钥，程序会弹出友好提示。

## 8. 完整功能演示流程
1. **导入日志**：点击「导入日志」，支持 `.log` / `.txt` / `.json` 格式日志文件；
2. **日志解析**：程序自动解析日志内容，状态栏展示导入状态与解析条数；
3. **AI 故障分析**：点击「AI 故障分析」，异步调用大模型执行智能诊断；
4. **查看结果**：右侧面板实时展示结构化分析报告；
5. **导出报告**：点击「导出报告」，将诊断结果保存为 `.md` / `.txt` 文件。

## 9. 项目亮点（技术 & 工程化）
1. **工业场景结合 AIGC**
   并非单纯内容生成，将大模型能力落地到工控上位机运维场景，贴合传统桌面软件业务。
2. **提示词工程 + 知识库约束**
   内置标准化故障知识库与强规则提示词，统一 AI 输出逻辑、格式、专业度，有效规避大模型幻觉问题。
3. **完善的工程化设计**
   全程异步网络请求，杜绝 UI 卡死；完善异常捕获、状态提示、按钮可用性管控，符合工业软件交互规范。
4. **密钥安全处理**
   采用 Windows 原生 DPAPI 加密存储密钥，禁止明文持久化，兼顾易用性与数据安全。
5. **高可配置、易扩展**
   提示词、知识库、接口参数全部以文件/配置形式外置，无需改代码即可适配不同大模型、不同设备场景。
